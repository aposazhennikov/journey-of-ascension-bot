# Journey of Ascension

Telegram-бот для постепенной внутренней практики на 4 языках: `ru`, `en`, `uz`, `kz`.

Бот помогает пользователю двигаться по двум направлениям:

- **Яма/Нияма** - нравственный фундамент практики. Бот ежедневно присылает принцип для наблюдения, чтобы постепенно внедрять его в жизнь.
- **Китайские меридианы** - следующий слой работы с вниманием, телом, дыханием и ощущением энергии. Пользователь может идти по рекомендованному пути или самостоятельно выбирать меридиан.

## Возможности

- 4 языка интерфейса и контента: русский, английский, узбекский, казахский.
- Выбор режима практики: только Яма/Нияма, только меридианы или оба направления.
- Отдельная логика для меридианов: маршрут бота или свободный выбор, текущий фокус, переход по точкам кнопками, список точек с возвратом к текущей практике, завершение меридиана после последней точки.
- 14 меридианов и 361 точка в `bot/meridians.json`.
- Изображения для точек и обзорные схемы меридианов в `images/meridians`.
- Справка по системе измерений в ТКМ: как отмерять `0,5`, `1`, `1,5`, `2`, `3` и `5` цуней.
- Справка по поиску точки: как уточнять расположение через изображение, цуни, касание, дыхание и внимание. Из справки по цуням можно сразу перейти к поиску точки, не перегружая первый экран меридианов.
- Карточки точек дают не только расположение, но и практическую подсказку по области тела: живот/таз, позвоночник, грудь, голова, рука или нога.
- Ежедневные напоминания в локальном времени пользователя.
- Настройка часового пояса, времени и дней без напоминаний.
- Главное меню с разделами Яма/Нияма, Меридианы, режимы практики, настройки, информация о боте и обратная связь.
- Локализованная админская рассылка анонса меридианов.
- HTML-симулятор без Telegram и автоматические UX-аудиты для текстов, кнопок, картинок, callback-логики и локалей.
- Быстрые сценарии HTML-симулятора для ручного прогона новичка: перед стартом меридианов и первая точка первого меридиана.
- Healthcheck, статистика, метрики и Sentry.

## Текущий статус

- Базовый MVP ветки меридианов реализован: все 14 меридианов, все 361 точки, изображения, локализация и навигация доступны в кодовой базе.
- Яма/Нияма остаётся фундаментом: ежедневная карточка принципа показывает часть практики, живой фокус дня и конкретное действие для наблюдения.
- Раздел меридианов уже поддерживает два пути: рекомендованный маршрут бота для новичков и свободный выбор конкретного меридиана.
- Карточки точек стали более телесными: блок концентрации теперь добавляет короткую подсказку по зоне, чтобы практика не ощущалась как один повторяющийся шаблон.
- Интро Переднесрединного и Заднесрединного меридианов выровнены на 4 языках: объясняют Инь/Ян, отсутствие фиксированной почасовой активности и практический смысл наблюдения центральной линии.
- Интро меридианов лёгких и толстой кишки выровнены на 4 языках: объясняют Инь/Ян, парность, направление движения энергии и связь с соседними меридианами.
- Интро меридианов желудка и селезёнки выровнены на 4 языках: показывают пару принятия и усвоения, направление движения энергии, связь с соседними меридианами и телесный фокус практики.
- Интро меридианов сердца и тонкой кишки выровнены на 4 языках: раскрывают тихий эмоциональный центр, навык различения, связь каналов и практику без драматизации ощущений.
- Первый экран меридианов сейчас намеренно компактный: продолжение текущей практики, явный старт/выбор пути изучения, справка по цуням и возврат в меню. Кнопка продолжения появляется только когда меридианная практика включена.
- Тексты и кнопки первого экрана меридианов упрощены: пользователь сразу видит, зачем изучать меридианы, как выбрать маршрут и где открыть справку по измерению цуней.
- Справочные экраны меридианов связаны между собой: цуни ведут к поиску точки, а кнопка текущего фокуса в справке появляется только при уже выбранном меридиане.
- HTML-симулятор сбрасывает состояние при открытии быстрых сценариев, чтобы ручная проверка не зависела от предыдущих кликов.
- Планировщик учитывает включённые режимы практики: старые пользователи мигрируют с включённой Ямой/Ниямой и выключенными меридианами, отключённые режимы не продолжают цепочку ежедневных заданий, а меридианное напоминание показывает текущий фокус без автоматического перехода к следующей точке.
- Ведётся редакторская полировка: тексты очищаются от сухих медицинских и источниковых оборотов, чтобы бот звучал как практический проводник, а не как справочник.

## Структура

```text

  main.py              # точка входа
  handlers.py          # команды, callbacks, меню и onboarding
  scheduler.py         # ежедневные задания
  storage.py           # JSON-хранилище пользователей и логов

  simulate_bot_ux.py   # HTML-симулятор UX и аудит данных
  render_bot_ux.py     # визуальный аудит UX
```

## Быстрый старт

1. Создайте тестового или продакшен-бота в [@BotFather](https://t.me/BotFather).
2. Скопируйте настройки окружения:

```bash
cp env.example .env
```

3. Заполните `.env`:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_IDS=123456789,987654321
DATA_DIR=data
HTTP_PORT=8080
LOG_LEVEL=INFO
ENVIRONMENT=production
```

4. Запустите контейнер:

```bash
docker compose up -d
docker compose logs -f yoga-bot
```

После изменения кода или Dockerfile можно пересобрать без кеша:

```bash
docker compose build --no-cache
docker compose up -d --force-recreate
```

## Тестирование без реальных пользователей

Для проверки новых сценариев используйте отдельного Telegram-бота и отдельную директорию `data`.

Рекомендуемый порядок:

1. Создать тестового бота в BotFather.
2. Указать его токен в `.env`.
3. Убедиться, что используется отдельная папка данных:

```env
DATA_DIR=data
```

4. Запустить контейнер и написать тестовому боту `/start`.

Если контейнер не может записывать `users.json` или `bot_messages.json`, проверьте права на `data`:

```bash
mkdir -p data
chmod -R 777 data
docker compose up -d --force-recreate
```

Для тестового сервера это самый быстрый способ убрать ошибку `Permission denied` на bind-mounted JSON-файлах.

## Команды пользователя

| Команда | Описание |
| --- | --- |
| `/start` | Запустить onboarding: язык, режим практики, часовой пояс, время, дни без напоминаний |

## Данные

Пользовательские данные лежат в `data/`:

- `users.json` - настройки, режимы практики, прогресс меридианов.
- `sent_logs.json` - журнал отправленных сообщений.
- `bot_messages.json` - сообщения, которые бот может удалять для чистого интерфейса.
- `feedback.json` - обратная связь.

Контент:

- `bot/principles.json` - 10 принципов Ямы и Ниямы.
- `bot/meridians.json` - меридианы, точки, локализации, источники, изображения.
- `images/` - изображения принципов.
- `images/meridians/` - схемы меридианов и точек.

## Docker volumes

В `docker-compose.yml` контент подключается так:

```yaml
volumes:
  - ./data:/app/data
  - ./images:/app/images:ro
  - ./bot/principles.json:/app/bot/principles.json:ro
  - ./bot/meridians.json:/app/bot/meridians.json:ro
```

Это позволяет обновлять контент и изображения без пересборки образа, если код не менялся.

## Важная продуктовая логика

- Старые пользователи остаются на Яма/Нияма, меридианы включаются только после выбора.
- Прогресс по точкам меридиана не двигается автоматически. Пользователь нажимает кнопки сам.
- Кнопка завершения меридиана появляется только на последней точке; старые карточки с прежней кнопкой не должны завершать меридиан раньше времени.
- Ежедневное сообщение по меридианам напоминает о текущем фокусе, но не переключает точку.
- Дни без напоминаний применяются ко всем активным ежедневным практикам.
- Бот не заменяет медицинскую диагностику или лечение. Меридианы используются как карта наблюдения, самодисциплины и внимательной практики.

## Troubleshooting

Проверить конфигурацию:

```bash
docker compose config
```

Посмотреть логи:

```bash
docker compose logs -f yoga-bot
```

Проверить, что Telegram-токен живой:

```bash
curl "https://api.telegram.org/bot<token>/getMe"
```

Если после `git pull` интерфейс не изменился, пересоздайте контейнер:

```bash
docker compose build --no-cache
docker compose up -d --force-recreate
```

Если менялись только JSON или изображения, обычно достаточно:

```bash
docker compose up -d --force-recreate
```

## Источники контента

Общая информация по меридианам и точкам собирается из согласованных источников пользователя. В приложении тексты адаптируются под формат медитативного изучения и локализуются на 4 языка.
