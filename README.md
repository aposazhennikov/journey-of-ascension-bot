# Journey of Ascension

Telegram-бот для постепенной внутренней практики на 4 языках: `ru`, `en`, `uz`, `kz`.

Бот помогает пользователю двигаться по двум направлениям:

- **Яма/Нияма** - нравственный фундамент практики. Бот ежедневно присылает принцип для наблюдения, чтобы постепенно внедрять его в жизнь.
- **Китайские меридианы** - следующий слой работы с вниманием, телом, дыханием и ощущением энергии. Пользователь может идти по рекомендованному пути или самостоятельно выбирать меридиан.

## Возможности

- 4 языка интерфейса и контента: русский, английский, узбекский, казахский.
- Выбор режима практики: только Яма/Нияма, только меридианы или оба направления.
- Onboarding объясняет новичку практическую логику пути: куда уходит энергия, как Яма/Нияма закрывает утечки через речь, привычки и реакции, и зачем меридианы нужны для телесного наблюдения Ци.
- Раздел Яма/Нияма уточняет, что принцип дня — это акцент для наблюдения, а не отмена остальных принципов.
- При выборе обоих направлений onboarding теперь собирает два отдельных времени: одно для Ямы/Ниямы и отдельное для меридианов.
- Четырёхшаговый onboarding для режима "оба направления" показывает честные счётчики `1/4`, `2/4`, `3/4` и объясняет, зачем меридианам отдельное время.
- Отдельная логика для меридианов: маршрут бота или свободный выбор, текущий фокус, переход по точкам кнопками, список точек с возвратом к текущей практике, завершение меридиана после последней точки.
- Первый экран меридианов сделан компактным: продолжить текущую практику, начать/выбрать путь изучения, открыть справку по цуням или вернуться в меню.
- Экран настроек показывает текущий ритм практики: активный путь, язык, часовой пояс, отдельные времена Ямы/Ниямы и меридианов, дни тишины. HTML-симулятор отображает этот snapshot тоже.
- 14 меридианов и 361 точка в `bot/meridians.json`.
- Изображения для точек и обзорные схемы меридианов в `images/meridians`.
- Standalone-аудит `scripts/audit_meridian_content.py` проверяет 14 меридианов, 361 точку, 4 локали, изображения и явные хвосты исходников без запуска Telegram-бота.
- Для готовых Переднесрединного и Заднесрединного меридианов обзорные схемы привязаны в `bot/meridians.json`, а все точки `VC1-VC24` и `VG1-VG28` имеют отдельные изображения.
- Справка по системе измерений в ТКМ: как отмерять `0,5`, `1`, `1,5`, `2`, `3` и `5` цуней. При открытии справки бот отправляет наглядную схему `images/meridians/cun_measurement.png`.
- Экран меридианов даёт новичку мягкое объяснение: что такое каналы Ци, почему точки могут ощущаться открытыми или «молчать», как один меридиан связан с соседними, и где граница между практикой самонаблюдения и медициной.
- Справка по поиску точки: как уточнять расположение через изображение, цуни, касание, дыхание и внимание. Из справки по цуням можно сразу перейти к поиску точки, не перегружая первый экран меридианов.
- Карточки точек дают не только расположение, но и практическую подсказку по области тела: живот/таз, позвоночник, грудь, голова, рука или нога.
- Полезные позиционные подсказки из источников больше не теряются: открыть рот, поднять руку, повернуть ладонь, согнуть колено, лечь на бок или на живот теперь показываются в блоке концентрации, а не как сухой хвост в расположении точки.
- Последняя точка каждого меридиана теперь подсказывает пройти вниманием весь канал целиком, чтобы завершение было практикой интеграции, а не просто нажатием кнопки.
- Подсказка "Проверьте область" корректно обрабатывает десятичные расстояния вроде `0.5 cun` и больше не превращает их в обрубок `0`.
- В английской и узбекской локалях названия точек отображаются латиницей, чтобы карточки и списки не смешивали алфавиты и не превращались в один технический код.
- Ежедневные напоминания в локальном времени пользователя.
- Настройка часового пояса, времени и дней без напоминаний.
- Главное меню с разделами Яма/Нияма, Меридианы, режимы практики, настройки, информация о боте и обратная связь.
- Локализованная админская рассылка анонса меридианов.
- HTML-симулятор без Telegram и автоматические UX-аудиты для текстов, кнопок, картинок, callback-логики и локалей.
- Быстрые сценарии HTML-симулятора для ручного прогона новичка: перед стартом меридианов и первая точка первого меридиана.
- Healthcheck, статистика, метрики и Sentry.
- `docker-compose.yml` использует актуальный Compose-формат без устаревшего поля `version`.

## Текущий статус

- Базовый MVP ветки меридианов реализован: все 14 меридианов, все 361 точки, изображения, локализация и навигация доступны в кодовой базе.
- Яма/Нияма остаётся фундаментом: ежедневная карточка принципа показывает часть практики, живой фокус дня и конкретное действие для наблюдения.
- Описания Астеи, Апариграхи, Шаучи и Свадхьяи вычитаны на 4 языках: убраны назидательные и сухие обороты, добавлен более практический фокус на внимание, энергию, честность и наблюдение за собой.
- Раздел меридианов уже поддерживает два пути: рекомендованный маршрут бота для новичков и свободный выбор конкретного меридиана.
- Карточки точек стали более телесными: блок концентрации теперь добавляет короткую подсказку по зоне, чтобы практика не ощущалась как один повторяющийся шаблон.
- Интро Переднесрединного и Заднесрединного меридианов выровнены на 4 языках: объясняют Инь/Ян, отсутствие фиксированной почасовой активности и практический смысл наблюдения центральной линии.
- Интро меридианов лёгких и толстой кишки выровнены на 4 языках: объясняют Инь/Ян, парность, направление движения энергии и связь с соседними меридианами.
- Интро меридианов желудка и селезёнки выровнены на 4 языках: показывают пару принятия и усвоения, направление движения энергии, связь с соседними меридианами и телесный фокус практики.
- Интро меридианов сердца и тонкой кишки выровнены на 4 языках: раскрывают тихий эмоциональный центр, навык различения, связь каналов и практику без драматизации ощущений.
- Интро меридианов мочевого пузыря и почек выровнены на 4 языках: описывают заднюю линию, глубокий ресурс, связь страха/отдыха/воли и соседние каналы без медицинских обещаний.
- Интро меридианов перикарда и трёх обогревателей выровнены на 4 языках: описывают здоровую защиту сердца, распределение тепла и внимания, а также связь с соседними каналами.
- Интро меридианов желчного пузыря и печени выровнены на 4 языках: раскрывают боковую линию, решение, свободное движение Ци и замыкание круга парных каналов.
- Автоаудит теперь ловит старые шаблонные интро меридианов, чтобы тексты не откатывались к роботизированному справочному стилю.
- Первый экран меридианов сейчас намеренно компактный: продолжение текущей практики, явный старт/выбор пути изучения, справка по цуням и возврат в меню. Кнопка продолжения появляется только когда меридианная практика включена.
- Тексты и кнопки первого экрана меридианов упрощены: пользователь сразу видит, зачем изучать меридианы, как выбрать маршрут и где открыть справку по измерению цуней.
- Справочные экраны меридианов связаны между собой: цуни ведут к поиску точки, а кнопка текущего фокуса в справке появляется только при уже выбранном меридиане.
- HTML-симулятор сбрасывает состояние при открытии быстрых сценариев, чтобы ручная проверка не зависела от предыдущих кликов.
- Пагинация списка точек стала человекочитаемой: кнопки показывают "Следующие 10" и "Предыдущие 10", а не технические `10`.
- Финальные пользовательские тексты меню, приветствия и завершения настройки переведены на HTML-разметку `<b>...</b>` без Markdown-остатков `**...**`; аудит теперь ловит возврат таких остатков в видимые сообщения.
- Планировщик учитывает включённые режимы практики: старые пользователи мигрируют с включённой Ямой/Ниямой и выключенными меридианами, отключённые режимы не продолжают цепочку ежедневных заданий, а меридианное напоминание показывает текущий фокус без автоматического перехода к следующей точке.
- Ведётся редакторская полировка: тексты очищаются от сухих медицинских, источниковых и слишком "ИИшных" оборотов, чтобы бот звучал как практический проводник, а не как справочник.
- Тексты "О боте" и первого экрана меридианов стали более живыми: меньше абстрактных формул, больше понятной связи с ежедневной практикой, вниманием, дыханием и ощущениями тела.
- Генератор `scripts/import_meridians.py` тоже очищен от старого шаблона интро, чтобы будущие обновления контента не возвращали механический тон.
- Описания Ахимсы, Сатьи, Брахмачарьи, Тапаса, Сантоши и Ишвара Пранидханы переписаны на 4 языках: вместо энциклопедических определений они объясняют, как принцип связан с расходом энергии, вниманием и конкретной практикой дня.
- Короткие подписи Астеи, Брахмачарьи, Апариграхи и Свадхьяи тоже очищены от словарного стиля: теперь они звучат как понятные практические ориентиры.
- HTML-симулятор теперь показывает изображения в карточках Яма/Нияма, поэтому ручная проверка без Telegram ближе к реальному поведению бота.
- Справка по поиску точки стала естественнее: пользователь получает простую инструкцию найти область по изображению и цуням, а затем уточнять точку ощущениями тела.
- Карточки точек теперь аккуратнее объясняют "закрытую" точку: это рабочий ориентир для практики внимания, массажа и дыхания, а не вывод о здоровье.
- Подпись к схеме цуня и справка "Как искать точку" переписаны проще: без абстрактных формулировок про карту/мишень, с прямым объяснением, как сначала найти область, а потом уточнять точку пальцами, дыханием и вниманием.
- Свободный выбор меридиана больше не перегружает экран всеми 14 каналами сразу: список разбит на 2 страницы по 7 меридианов, это же поведение проверяется в браузерном симуляторе.
- Экран "Все точки" тоже стал спокойнее: длинные меридианы показывают по 7 точек на страницу, без повторения номера страницы в тексте и без кнопок вида "Следующие 10".
- В карточках точек блок "Наблюдение" стал короче и больше не повторяет координаты точки; автоаудит проверяет, что после подгонки под Telegram-caption вопрос наблюдения не пропадает.
- Базовые инструкции всех 361 точек обновлены на 4 языках: первая точка учит найти одно ощущение, а следующие точки добавляются к уже найденной линии меридиана. Реальный formatter, HTML-симулятор и preview теперь берут эту практику из `meridians.json`, а не из отдельного шаблона.
- Финальное сообщение настройки стало менее механическим: вместо "бот будет..." оно описывает выбранный ритм практики и напоминает, что остальные принципы не выключаются даже при режиме Яма/Нияма + меридианы.
- Автоаудит проверяет, что в текстах не возвращаются старые шаблонные, энциклопедические и слишком сгенерированные обороты; также проверяется наличие схемы измерения цуня.

## Быстрый старт

1. Создайте тестового или продакшен-бота в [@BotFather](https://t.me/BotFather).
2. Скопируйте настройки окружения:

```bash
cp env.example .env
```

3. Заполните `.env`:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_IDS=123456789,987654321
DATA_DIR=data
HTTP_PORT=8080
LOG_LEVEL=INFO
ENVIRONMENT=production
```

4. Запустите контейнер:

```bash
docker compose up -d
docker compose logs -f yoga-bot
```

После изменения кода или Dockerfile можно пересобрать без кеша:

```bash
docker compose build --no-cache
docker compose up -d --force-recreate
```

## Тестирование без реальных пользователей

Для проверки новых сценариев используйте отдельного Telegram-бота и отдельную директорию `data`.

Рекомендуемый порядок:

1. Создать тестового бота в BotFather.
2. Указать его токен в `.env`.
3. Убедиться, что используется отдельная папка данных:

```env
DATA_DIR=data
```

4. Запустить контейнер и написать тестовому боту `/start`.

Если контейнер не может записывать `users.json` или `bot_messages.json`, проверьте права на `data`:

```bash
mkdir -p data
chmod -R 777 data
docker compose up -d --force-recreate
```

Для тестового сервера это самый быстрый способ убрать ошибку `Permission denied` на bind-mounted JSON-файлах.

## Локальный UX-симулятор

Симулятор помогает проверять тексты, кнопки, изображения и сценарии без Telegram.

```bash
py -X utf8 scripts\simulate_bot_ux.py
py -m http.server 8766
```

Откройте:

```text
http://127.0.0.1:8766/tmp/ux_simulator.html
```

Можно открывать конкретный экран сразу:

```text
http://127.0.0.1:8766/tmp/ux_simulator.html?lang=ru&scenario=noviceMeridians
http://127.0.0.1:8766/tmp/ux_simulator.html?lang=ru&scenario=noviceFirstPoint
```

Важно: запускайте `http.server` из корня проекта. Если отдавать только папку `tmp`, HTML откроется, но изображения из `images/meridians` не загрузятся.

Для ручного UX-прогона новичка используйте быстрые сценарии:

- `Novice: before meridian start` - вход в раздел меридианов до выбора маршрута.
- `Novice: first meridian point` - первая точка первого меридиана с изображением и практикой.

Аудиты:

```bash
py -X utf8 scripts\simulate_bot_ux.py --audit-only
py -X utf8 scripts\render_bot_ux.py --audit-only
py -X utf8 scripts\audit_meridian_content.py
py -m compileall .\bot .\scripts
git diff --check
```

## Данные

Пользовательские данные лежат в `data/`:

- `users.json` - настройки, режимы практики, прогресс меридианов.
- `sent_logs.json` - журнал отправленных сообщений.
- `bot_messages.json` - сообщения, которые бот может удалять для чистого интерфейса.
- `feedback.json` - обратная связь.

Контент:

- `bot/principles.json` - 10 принципов Ямы и Ниямы.
- `bot/meridians.json` - меридианы, точки, локализации, источники, изображения.
- `images/` - изображения принципов.
- `images/meridians/` - схемы меридианов и точек.

## Docker volumes

В `docker-compose.yml` контент подключается так:

```yaml
volumes:
  - ./data:/app/data
  - ./images:/app/images:ro
  - ./bot/principles.json:/app/bot/principles.json:ro
  - ./bot/meridians.json:/app/bot/meridians.json:ro
```

Это позволяет обновлять контент и изображения без пересборки образа, если код не менялся.

## Важная продуктовая логика

- Старые пользователи остаются на Яма/Нияма, меридианы включаются только после выбора.
- Прогресс по точкам меридиана не двигается автоматически. Пользователь нажимает кнопки сам.
- Кнопка завершения меридиана появляется только на последней точке; старые карточки с прежней кнопкой не должны завершать меридиан раньше времени.
- Ежедневное сообщение по меридианам напоминает о текущем фокусе, но не переключает точку.
- Дни без напоминаний применяются ко всем активным ежедневным практикам.
- Бот не заменяет медицинскую диагностику или лечение. Меридианы используются как карта наблюдения, самодисциплины и внимательной практики.

## Troubleshooting

Проверить конфигурацию:

```bash
docker compose config
```

Посмотреть логи:

```bash
docker compose logs -f yoga-bot
```

Проверить, что Telegram-токен живой:

```bash
curl "https://api.telegram.org/bot<token>/getMe"
```

Если после `git pull` интерфейс не изменился, пересоздайте контейнер:

```bash
docker compose build --no-cache
docker compose up -d --force-recreate
```

Если менялись только JSON или изображения, обычно достаточно:

```bash
docker compose up -d --force-recreate
```

## Источники контента

Общая информация по меридианам и точкам собирается из согласованных источников пользователя. В приложении тексты адаптируются под формат медитативного изучения и локализуются на 4 языка.
